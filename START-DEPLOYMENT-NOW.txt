  - Logs saved to: /tmp/deal_tracking_install_*.log
  - Safe to interrupt (backups available)

If Deployment Fails:
  1. Check SSH connection:
     ssh root@139.84.163.11
  
  2. Check Odoo service:
     systemctl status odoo
  
  3. View detailed logs:
     tail -100 /var/log/odoo/odoo-server.log | grep -i error
  
  4. Check file permissions:
     ls -la /var/odoo/scholarixv2/extra-addons/*/commission_ax/models/
  
  5. If critical issue, rollback:
     systemctl stop odoo
     cp -r backups/commission_ax_YYYYMMDD_HHMMSS/* \
           extra-addons/odooapps.git-68ee71eda34bc/commission_ax/
     systemctl start odoo

If Fields Don't Appear After Upgrade:
  1. Refresh browser (Ctrl+Shift+R for hard refresh)
  2. Clear Odoo cache in database:
     rm -r ~/.cache/odoo*
  3. Restart Odoo:
     systemctl restart odoo
  4. Reload Odoo web interface

If Tests Fail:
  1. Verify module upgraded successfully
  2. Check user has appropriate permissions
  3. Verify database connection
  4. Check browser developer console for JS errors
